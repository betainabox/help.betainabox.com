Getting Started

To start prototyping your application, start by selecting a recipe from our library.  A “recipe” is a template for your application. It can contain the web resources to be shown to the user (e.g., the HTML files, the stylesheets and images) as well as predefined “sauces” which describe the behaviour of the application.  You can customize the look and feel of the recipe in our Recipe Editor, and you can upload your own recipe as well.


Once the recipe is ready, you can start adding behaviour to it by adding “sauces.”  A sauce is a combination of a trigger and a series of activities. It is a complete workflow that describes the starting condition (the trigger) and the actions to be taken when the condition happens (the activities)


When you are ready to have your users test the application, go to the "Serve" page where you can customize aspects of your application (e.g., change the web site address, and change the description that shows up when a search engine finds your application) You can also enable services such as collecting visitors analytics and user feedback.
